## Rendu attendu & planning

1. **Avant fin octobre** (le plus tôt le mieux) : Vous devez me faire valider votre application, en m'envoyant un email avec des informations sur votre choix, ainsi que les personnes de votre groupe.
2. **Tout au long du projet**, toutes les 2 semaines : vous devez m’envoyer un compte rendu par mail ou sur une carte Taiga (ou autre logiciel de gestion de projet) de la progression de votre projet.
3. **Une semaine avant le dernier cours du semestre** (au plus tard) : Vous devez me donner un accès à votre dépôt GitLab, GitHub (ou autre) pour que je puisse commencer à étudier votre code.
3. **La veille du dernier cours** (au plus tard) : Votre rapport PDF doit être déposé sur Moodle

### Rapport

:::danger[Important]
Une partie de la note sera basée sur la qualité de votre rapport : présentation, clarté, structure, etc.
:::

* Écrivez un rapport de au moins 10 pages présentant votre application, les algorithmes et résultats de vos tests et conclusions, les difficultés rencontrées, et un guide rapide d’utilisation de l’interface.

* Une partie du rapport doit présenter votre gestion de projet (Taiga, Trello, planning, etc.) et les tâches réalisées par chaque membre du groupe.

* En annexe, vous **devez** :

  * préciser toutes les sources d’information (article scientifique, blog, tutoriel) et librairies (graphiques, d’algorithmes, etc.) que vous avez consultées ou utilisées pour la réalisation de votre projet.

* Le fichier du rapport doit être un fichier PDF, et le nom du fichier doit contenir les noms de famille des personnes du groupe ( Par exemple `IA_Projet_NOM1_NOM2_NOM3.pdf` ).

</Slide>
<Slide>

### Code source

:::danger[Important]
Une partie de la note sera basée sur la qualité de votre code source : structure, commentaires, tests, etc.
:::

* Le code source de votre application doit être disponible sur un dépôt [GitLab](https://about.gitlab.com/) ou [Github](https://github.com/). Pour info, l'UGA propose des dépôts [GitLab](https://gitlab.univ-grenoble-alpes.fr/) pour les étudiants.

* Votre dépôt **doit être privé**.

* Vous devez me donner accès à votre dépôt en m’ajoutant : [pierre.blarre@univ-grenoble-alpes.fr](mailto:pierre.blarre@univ-grenoble-alpes.fr) 

* Le fichier readme.md de votre dépôt **doit contenir** des instructions d’installation de votre projet : (Langages et librairies utilisées \+ instructions d’installation (git clone…, npm install, etc.))

* Testez l’installation et l'exécution de votre votre application sur différents OS (Windows, Linux, MacOS) et navigateurs (Si votre application est web).

* Si vous utilisez une solution technique avec des dépendances (NodeJs / NPM, PHP/Composer, pip3, cargo, etc.), pensez à les exclure de votre Git et bien rédiger dans vos instructions d’installation comment procéder.

* Vous devez préciser comment lancer les tests qui vous ont permis de générer des statistiques pour que je puisse les exécuter de mon côté.

</Slide>

## Gestion de projet 

:::danger[Important]
Une partie de la note sera basée sur votre gestion de projet : organisation, planning, respect des deadlines, etc.
:::

* Vous pouvez dupliquer ou vous inspirer de [ce tableau Taiga](https://tree.taiga.io/project/erralb-uga-projet-ia/kanban) pour organiser votre projet ou bien utiliser un autre outil de votre choix.

* Prévoyez dès maintenant des deadlines régulières de tâches à compléter. 

* Comptez au grand minimum 2h de travail par semaine par personne pour réussir le projet.

* Vous devez m’envoyer un compte rendu par mail ou sur une carte Taiga de la progression de votre projet **toutes les 2 semaines**. Vous perdrez des points si vous ne respectez pas cette règle.
